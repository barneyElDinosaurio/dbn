Summary

The Design By Numbers Courseware is an experimental work-in-progress. A web-based environment built to facilitate the use of Design By Numbers as an educational tool. It provides educators with an interface for inputting/editing problem statements and students with the ability to work in Design By Numbers and save their work directly to the server. Most importantly it presents a unique viewing environment where students can see each other's work in progress and critique their work within the context with the entire class. To date, the Courseware has been used in both formal classroom settings and informal seminars.

The Courseware is a group of CGI scripts written in Perl. Individuals/institutions planning on using the Courseware should have access to both Perl and a web-server (either WebTen for Mac, or Apache for PC and UNIX).

It is highly recommended that the Design By Numbers Courseware is used with 4.0+ browsers.

Additional information about Design By Numbers can be found at "http://dbn.media.mit.edu".

Configure / Making it work

Setting up the Courseware requires some knowledge about web servers and CGI scripts. The Courseware has been tested with the following configurations: Mac OS8/WebTen Server, Windows98/Apache/Perl, Digital Unix/Apache/Perl. Before configuring the Courseware, make sure your server and Perl are both working.

The Courseware is comprised of three primary directories (dbn, courses, and images) and a set of CGI scripts. The Courseware download includes all of the CGI scripts in a directory called 'cgi-bin'. The 'dbn' directory includes the java classes and text files required to run Design By Numbers. The 'courses' directory includes only a sample problem set as a text file, but will include all of the course data (see Appendix C) as more courses are added. The 'images' directory includes the four images that the courseware uses.


There are six basic tasks involved in setting up the Courseware.

1. Uncompress the file you downloaded (either 'dbncourseware.zip' for unix/windows or 'dbncourseware.sea.bin' for macintosh.)

2. Copy the directories and their contents onto your webserver. The major decisions that need to be made are the locations of the CGI scripts and the other primary directories. I think a good organization is to place the 'dbn', 'courses', 'images', and 'cgi-bin' directories into a directory called 'dbncourseware', which is in the root level of the server. (Consult Appendix A for more information)

3. Input directory structure and URL information into 'globals.pl'. Edit 'globals.pl' in a text editor and update the path and URL information at the top of the file. This will set up the relationship between the CGI scripts, the Design By Numbers applet, and the course data.

4. Update the CGI scripts so that they can find Perl. To do this change the first line of all the '.cgi' files to reflect Perl's location on the server. For example: #!/usr/bin/perl is a likely target on Unix and #!/perl/bin/perl would point to c:\perl\bin\perl.exe on windows.

5. If your web server is running on a Unix variant, make sure that the permissions for your directories all allow appropriate access. Specifically, make sure the '.cgi' scripts are executable by everyone (chmod 755) and the 'courses' directory is writable by everyone (chmod 777).

6. Alter the 'index.html' file to redirect to the location of 'login.cgi' on your server. Put this file in any directory in which you wish to obscure its contents.


To test and see if everything is working, try opening 'create.cgi' in your web browser.

Creating a course

To create a course, open the 'create.cgi' file in your web browser and follow the instructions. If you receive an error message during the creation of the course, your CGI scripts are either not configured correctly or not compatible with your hardware/software configuration.

Appendix B -- The CGI Scripts

The Design by Numbers Courseware is comprised of twelve CGI scripts and a Perl file that contains functions and data common to all of the scripts.
create.cgi   accepts info required to create the course, writes data files and directories
problems.cgi   deletion and creation of problem sets
editproblems.cgi   facilitates creation/editing of problem sets, sends updates to 'updateprob.cgi'
people.cgi   edit/delete students
home.cgi   different for each category of user. guests can chose what problems they want to see. students can choose to view problems or work. the admin can select the edit features.
showproblems.cgi   displays the problem statements
view.cgi   displays solutions to problem sets
login.cgi   user inputs name and password, links to 'home.cgi' if information is valid
selectproblem.cgi   student selects the problem they want to work on
work.cgi   displays the DBN programming environment, user can save work to server
globals.pl   contains global data, global functions
fetch.cgi   retrieves file information
updateprob.cgi   saves changes to problems in 'problems.txt'


There are three different categories of users for the Courseware: the course administrator, the students, and guests. Below is a table decribing each class of users' permissions.

A   create.cgi
A   problems.cgi
A   editproblems.cgi
A   people.cgi
A   updateprob.cgi

A, S, G

   home.cgi
A, S, G   showproblems.cgi
A, S, G   view.cgi
A, S, G   fetch.cgi
A, S   login.cgi
A, S   selectproblem.cgi
A, S   work.cgi

A = pages accessible to or used by the 'A'dministrator
S = pages accessible to or used by 'S'tudents
G = pages accessible to or used by 'G'uests

Appendix C -- Course Data

The data for each course you create is stored in that course's directory. Each course has three data files:
admin.txt -- contains the course title, user names, and passwords
people.txt -- a list of the people in the course
problems.txt -- an annotated list describing all of the assignments

In addition, each student has their own directory into which their work is stored. When a student saves their work, it is stored as a text file with a '.dbn' extension and as a '.pgm' image.
