This is the documentation for all the PARAM tags used by the DBN applet.
(last updated 9.25.99 by ben fry)


mode editor - default, shows text area
grid - gridded layout used for courseware and the rest
 editor
inline_program paper 50; line 20 10 90 80; refresh
put an entire dbn program directly into the html to have it automatically load use semicolons for newlines it will be auto-beautified after loading 		none
program filename.dbn, attempts to load from:
  1. (application only) it attempts to load the file straight from the disk, either using a path relative to the app, or an absolute path
  2. (applet only) relative to the document base -- the same directory as html file
  3. (application only) as a 'resource', which would be a file in the same directory as the classes, just like codebase for an applet
  4. (applet only) relative to the codebase -- i.e. dbn.jar or 'classes' folder
  5. as an actual url (http://blahblahblah)

language one of the ISO language codes, for example: en - english, jp - japanese, fr - french, es - spanish, de - german
this will determine how programs should be translated when run		 en (no translation)
encoding 8859_1, SJIS
a text encoding converter, that will be used to convert any files read by the applet. these are based on sun's goofy naming specification, which you can find out more about by looking up java.lang.String in the Java Class Libraries vol. 1, 2nd edition. 		8859_1 (platform and locale specific)
